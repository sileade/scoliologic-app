# Клиентское приложение Scoliologic.ru

Клиентское приложение для пациентов ортопедического центра Scoliologic.ru. Приложение предоставляет пациентам доступ к их медицинским данным, планам лечения, а также обеспечивает связь с лечащими врачами.

## Ключевые возможности

*   **Личный кабинет пациента:** Просмотр и редактирование личной информации.
*   **План лечения:** Отслеживание прогресса реабилитации, выполнение упражнений и заданий.
*   **База знаний:** Доступ к статьям и видео по лечению и реабилитации.
*   **Запись на прием:** Удобная запись на консультации и процедуры.
*   **Документы:** Хранение и просмотр медицинских документов (рентгены, заключения и т.д.).
*   **Мои изделия:** Информация о протезах, ортезах и корсетах, полученная из МИС.
*   **Защищенный мессенджер:** Безопасное общение с врачами и специалистами клиники.

### UX/UI принципы

- **Mobile-first** — оптимизация для мобильных устройств
- **Интуитивная навигация** — нижняя панель с основными разделами
- **Скруглённые элементы** — дружелюбный интерфейс (border-radius: 12-24px)
- **Крупные touch-зоны** — минимум 48px для кнопок
- **Анимации** — плавные переходы и микроинтеракции

## Технологический стек

| Категория | Технология |
|-----------|------------|
| **Frontend** | React, TypeScript, Tailwind CSS, Vite |
| **Backend** | Node.js, Express, tRPC |
| **База данных** | MySQL, Drizzle ORM |
| **Аутентификация** | JWT, OAuth 2.0 (интеграция с Госуслугами) |

| **Контейнеризация** | Docker, Docker Compose |

## Структура проекта

Проект использует монорепозиторий со следующей структурой:

```
/scoliologic-app
├── client/         # Frontend (React)
│   ├── src/
│   │   ├── components/   # UI компоненты
│   │   ├── pages/        # Страницы приложения
│   │   ├── contexts/     # React контексты
│   │   └── lib/          # Утилиты и хелперы
├── server/         # Backend (Node.js, Express, tRPC)
├── drizzle/        # Схема и миграции базы данных
└── shared/         # Общие типы и константы
```

## Начало работы

1.  **Установите зависимости:**

    ```bash
    pnpm install
    ```

2.  **Настройте переменные окружения:**

    Скопируйте `docs/env-template.txt` в `.env` и заполните необходимые значения.

3.  **Запустите базу данных:**

    ```bash
    docker-compose up -d
    ```

4.  **Примените миграции базы данных:**

    ```bash
    pnpm db:push
    ```

5.  **Запустите приложение в режиме разработки:**

    ```bash
    pnpm dev
    ```

## Функции в разработке

### 1. Интеграция с Госуслугами (ЕСИА)

Реализация регистрации и входа через Единую систему идентификации и аутентификации (ЕСИА). Это упростит доступ для пациентов и повысит безопасность.

**Статус:** В разработке

### 2. Интеграция с МИС

Модуль для интеграции с медицинской информационной системой (МИС) клиники. Позволит автоматически получать и отображать информацию о назначенных изделиях (протезы, ортезы, корсеты).

**Статус:** Планируется

### 3. Защищенный мессенджер

Встроенный мессенджер для общения пациентов с врачами с использованием сквозного шифрования (E2EE) для обеспечения конфиденциальности переписки.

**Статус:** Базовая реализация готова
